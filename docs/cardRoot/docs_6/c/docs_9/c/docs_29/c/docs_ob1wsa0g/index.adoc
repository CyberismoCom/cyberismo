== Parameters

* `Key`: An identifier of an object. For example, a card key.
* `Field`: A name of a field
* `Value`: The calculated value of the field

== Description

The `fieldCalculated` predicate declares the automatically calculated value of a field.

Use `fieldCalculated` in the heads of rules that define calculated fields. Cyberismo derives the effective `{{#xref}}"cardKey":"docs_8mg8p068"{{/xref}}` value from it: if the user has entered an override for the field (see `{{#xref}}"cardKey":"docs_6h0t096b"{{/xref}}`), the override becomes the value of `field`; otherwise the `fieldCalculated` value does.

We recommend using `fieldCalculated` for all calculated fields. A plain `field` head still works when the field does not enable override, but only `fieldCalculated` keeps the automatic value visible when an override is in effect, and it is required for fields that enable override — otherwise the field ends up with conflicting values.
